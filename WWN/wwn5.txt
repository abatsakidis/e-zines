          Why are CD-RIP programs released over and over again?  Sometimes
       they are packaged poorly, sometimes they are missing files,
       sometimes they are too big and sometimes they don't work properly.
       What's so difficult?  What are these mistakes repeated?  All you
       have to do is ARJ *.* -v 1440 to compress the CD into 1.44 megabyte
       volumes.  Then you simply add your NFO files and FILE_ID.DIZ then
       zip each volume using PKZIP.  That may seem fool-proof, but there
       are a lot more complexities behind this.  For example, the
       Mechwarrior 2 CD release by OTFW required 2 install fixes and 2
       hacked EXE files to get it working properly.  The TDU release
       worked perfectly fine.  Where's the gap?  TDU made a loader and
       properly packaged the game, while OTFW was missing a crucial hidden
       file and included a CD emulator that simply did not work with the
       game.  Making loaders and fully testing the CD releases is a great
       idea.  You just can't lose when your release is ready to use after
       unzipping.  Some attempts at installation programs or batch files
       worked, but unless the job is done properly, you will end up with a
       mess of files on your hand.  If PKZIP was as flexible as ARJ and
       could create multiple volumes in the same manner, that would
       eliminate the need for archives insides archives.  Some ZIP chunker
       programs have come close, but ARJ remains as the standard for
       ripping CD's.  Learn from past mistakes.  Make sure you do not miss
       any files when creating your ARJ files.  Create a simple batch or
       script that installs the program and/or runs it.  Make sure the CD
       emulators work properly with the program all the way through.
       Don't release a CD rip (or full) too big.  It is very bothersome to
       deal with large releases.  If you plan on releasing something big,
       do it in parts.  For example, release an animation pack or
       Sound/Speech patch separate from the main CD rip.  This way, users
       with limited space can still use the programs.  You must keep in
       mind that the installation process takes up a lot more space than
       the final product.  The ZIP files, the ARJ files and the
       installation files must all be there.  In the end, fully testing
       and double checking everything pays off.  No more re-releases due
       to poor packaging, and no more faulty releases.  Do your homework,
       test the files, and keep the end user in mind.
